# 模型体系说明

## 模型不是现实本身，而是分析模板

一个好的模型满足三个要求：

1. **抽象性**：能适用于很多对象，不是只适用于一个案例。
2. **可推导性**：结合公理，可以自然推出大量结论。
3. **可扩展性**：遇到新对象（如未来的 AI 社会），只需继承已有模型，无需重新发明理论。

---

## 继承关系

```text
System（系统）
├── Evolution（演化）── 所有模型都遵守的通用模式
├── LifeSystem（生命系统）
│   └── Human（个人模型）
└── SocialSystem（社会系统）
    ├── Organization（组织模型）
    │   └── Nation（国家模型 = 组织 + 领土）
    └── Market（市场模型）
```

---

## 统一对象映射

分析任何问题时，第一句话先建模：

| 研究对象 | 模型归属 |
|----------|----------|
| 婚姻 | Social System |
| 公司 | Organization |
| 国家 | Organization + Territory |
| AI Agent | Life System（弱） |
| 人 | Human |
| 货币 | Market 中的资源 |
| 教育 | Social System |
| 宗教 | Social System |
| 军队 | Organization |
| 家庭 | Organization（小） |

---

## 使用方式

```text
1. 确定研究对象
2. 匹配模型（继承链最近的那个）
3. 用模型的属性框架展开分析
4. 结合公理进行推导
5. 得出结论
```

这样，你构建的就不是一堆观点，而是一套**统一建模语言**。
